1. Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделано предварительно?

git clone работает как обёртка над некоторыми другими командами. Она создаёт новый каталог, переходит внутрь и выполняет git init для создания пустого репозитория, затем она добавляет новый удалённый репозиторий (git remote add) для указанного URL (по умолчанию он получит имя origin), выполняет git fetch для этого репозитория и, наконец, извлекает последний коммит в ваш рабочий каталог, используя git checkout.


2. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

git init - Чтобы превратить обычный каталог в Git репозиторий и начать версионировать файлы в нём.
Создать новый репозиторий на GitHub.com
git remote add origin <REMOTE_URL> - добавить новый удаленный репозиторий
git push origin main - Отправим изменения в локальном репозитории в GitHub.com.


3. Как получить на локальный компьютер изменения с github.com

git fetch - для получения новых работ, выполненных другими людьми.
git pull — это удобное сочетание клавиш для выполнения git fetch и git merge в одной команде.


4. Какую информацию нам дают команда :

git status -отображает состояние рабочего каталога и раздела проиндексированных файлов. С ее помощью можно проверить индексацию изменений и увидеть файлы, которые не отслеживаются Git

git log -  используется для просмотра истории коммитов, начиная с самого свежего и уходя к истокам проекта.

git remote -v - показывает cписок удаленных подключений к другим репозиториям  и их URL-адресса
